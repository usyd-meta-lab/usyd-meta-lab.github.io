How we work

  • Communication: Microsoft Teams channels for groups; DM for quick checks.
  • Meetings: Weekly lab meeting; 1:1s as needed. Larger multi-lab meetings focused on either cognition (SLAC) or individual differences (METALICA) are available
  • Project documentation: README per project.
Always confirm ethics scope covers your manipulation and sample before recruitment.

How to choose a research question?

Selecting a research question can be difficult and usually projects evolve and change over time. The checklist below to quickly shape a viable, interesting project.

  • Sources of ideas: recent papers and reviews, replication + extension of a classic effect, open datasets, the lab’s example paradigms, or a method you want to learn.
  • Fit to lab themes: metacognition, learning, decision-making, confidence, individual differences, online experimentation.
  • Feasibility by degree: Honours: 1–2 experiments, n≈120 hours online; Masters: 1–2 studies (made up of multiple experiments); PhD: program of studies with clear phases.
  • Practical constraints: recruitment (SONA vs Prolific), incentives/budget, stimuli availability, and timeline.
  • Ethics scope: confirm you’re covered by an existing protocol or note amendments needed early (especially for Honours).
For Honours students, we normally focus on a specific theme each year and discuss target papers within the theme over the first few lab meeting of the year.

Next steps: read ~5 recent papers + 1 foundational review, sketch the task(s), check ethics coverage with Kit, and plan a small pilot.

Research Roadmap

The Research Roadmap outlines the typical lifecycle of a project in the lab, from design through coding, data collection, analysis, and decisions about replication, writing, or stopping. It helps set expectations about timelines (often ~2 years from idea to publication) and emphasises efficiency, replication, and the idea of a minimum viable experiment (MVE).

How to set up a project?

Projects follow a standard structure to keep code, analyses, and outputs tidy and versioned.

Project_Name/
                    ├── README.md
                    ├── experiments/
                    │   └── my-experiment/
                    │       ├── scripts/              # jsPsych task, stimuli, css
                    │       ├── data/                # raw/ processed/ metadata/
                    │       └── analysis/            # R scripts, notebooks, reports
                    ├── manuscripts/                 # writing
                    ├── preregistration/             # AsPredicted/ OSF
                    └── presentations/               # slides, posters
Analysis and data live inside each experiment folder. Use the lab R helper to scaffold: MetaLab::create_MetaLab_project(name, experiment, dir)

How to build a jsPsych experiment?

  • Start from template; keep files under assets/js, assets/css, assets/img.
  • Include browser/fullscreen checks and summary trials for easy QC.
  • Store key fields: participant_ID, source, trial_type, blocknum, trialnum, correct, confidence.
  • See example paradigms in our task library.

More details: You can view community tutorials on coding with jsPsych

How to launch a study?

We run browser-based experiments using jsPsych deployed on GitHub Pages with data stored via DataPipe to OSF. Follow our detailed walkthrough:

  • Build from the lab jsPsych template or an existing task from our task libraryrecommended
  • Set up OSF and DataPipe, then link them.
  • Host your study via GitHub Pages and test with URL parameters.
  • Configure automatic redirects for SONA or Prolific completion.

Read the step-by-step guide: Launching Studies

Tip: pilot with at least 3 people and check the browser console for issues before recruiting.

How to request SONA or Prolific access?

  • SONA: Request an account via the School’s process (see link in Putting the study on SONA).
  • Prolific: Share your Prolific username with Kit to join the lab team for access to funds.
  • Ethics: Ensure you are added to a lab ethics protocol before any data collection.
  • Redirects: Collect your SONA experiment ID + token and Prolific completion codes to configure in your study.

See: Prolific · SONA · Redirect Links

How to collect and store data?

  • DataPipe sends jsPsych data directly to your OSF project.
  • Use clear study IDs and record recruitment source for each participant.
  • For local analysis, download CSVs from OSF or use the lab R function to combine them. See here to set up authentication.
  • If you need to book the lab testing space, use LabArchive, see the School intranet
R helper: MetaLab::combine_osf_csvs("OSF_PROJECT", folder = "../Data/")

How to analyse data?

  • Use R & RStudio; keep a clean pipeline with (01_cleaning.R that produces a clean and publishable dataset then 02_analysis.R performs the analysis).
  • Within the my-experiment/data/metadata folder store a data dictionary. A template dictionary will be auto generated by the R function MetaLab::create_MetaLab_project(name, experiment, dir).
  • Use simulated or pilot data to lock analysis before full data.

See Resources for the lab package and starters.

How to write and publish?

  • Draft in Word; store in manuscripts/. Don't create multiple versions - OneDrive has built in versioning.
  • Maintain an up-to-date references library (use the EndNote shared library).
  • Share preprints and code/data on OSF where appropriate.
